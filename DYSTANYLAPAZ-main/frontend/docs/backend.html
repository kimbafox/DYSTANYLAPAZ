Arquitectura del Backend

El servidor fue reorganizado en módulos para separar responsabilidades.

Estructura

  • server/src/app.js: composición principal de Express.
  • server/src/routes: rutas por dominio.
  • server/src/middleware: autenticación y CORS.
  • server/src/services: lógica de sesiones y propiedades.
  • server/src/data/store.js: persistencia en data/db.json.
  • server/src/utils: validaciones, seguridad y seed inicial.

Persistencia

La base actual es un archivo JSON local. Es adecuada para desarrollo, demo y pruebas funcionales. Para producción el siguiente paso natural es migrar a PostgreSQL o MySQL.

API disponible

  • POST /api/auth/register
  • POST /api/auth/login
  • GET /api/auth/me
  • POST /api/auth/logout
  • GET /api/users
  • DELETE /api/users/:id
  • GET /api/properties
  • POST /api/properties
  • DELETE /api/properties/:id
